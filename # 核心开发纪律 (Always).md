# 核心开发纪律 (Always)

## 1. 技术栈约束 (Tech Stack)
- **框架：** 必须使用 Vue 3 (Composition API) + TypeScript + Vite。
- **动画：** 动效交互必须优先使用 **GSAP**，禁止编写复杂的原始 CSS Animation。
- **渲染：** 游戏核心逻辑（网格渲染、方块移动）必须在 **Canvas** 中完成，UI 层（分数、按钮、弹窗）使用 Vue 组件。
- **样式：** 使用 Tailwind CSS 进行布局，严禁使用任何具有尖锐棱角的 UI 设计。

## 2. 视觉与交互灵魂 (The Vibe)
- **圆角法则：** 所有的 UI 元素、Canvas 绘图、弹窗必须具有极大的圆角（例如 `border-radius: 24px` 或以上）。
- **Q弹物理：** 方块触底、旋转、消除时，必须调用 GSAP 实现 `Elastic` 或 `Back` 类型的缓动效果，模拟果冻/软糖质感。
- **动物形象：** 如果没有外部图片，请通过 Canvas 绘图指令绘制：带圆角的矩形 + 两个黑色小圆点（眼睛）+ 倒三角形（鼻子）。
- **配色准则：** 严格遵循莫兰迪糖果色系（粉蓝、粉紫、奶油黄、薄荷绿）。

## 3. 代码架构要求 (Architecture)
- **逻辑分离：** - `GameCore.ts`: 纯逻辑类，处理二维数组矩阵、旋转矩阵算法、消行逻辑。
    - `Renderer.ts`: 负责 Canvas 绘图逻辑，接收 GameCore 的状态进行逐帧渲染。
    - `useControls.ts`: 封装键盘与移动端滑动/点击手势的监听。
- **性能优化：** 必须使用 `requestAnimationFrame` 驱动游戏循环，严禁使用 `setInterval` 处理物理帧。
- **文件长度：** 单个文件严禁超过 250 行，超过逻辑需进行模块化拆分。

## 4. 专属礼物细节 (Gift Experience)
- **情感化提示：** 所有的系统提示语（如“游戏结束”、“准备好了吗”）必须使用可爱、治愈的语气。
- **彩蛋逻辑：** 预留分数监听接口，当分数达到 520, 1314 等特殊数字时，必须触发全屏爱心粒子特效（使用 Confetti 逻辑）。

## 5. 语言与注释 (Language)
- **交互语言：** 游戏界面文字必须使用中文。
- **注释要求：** 关键算法（如方块旋转时的边界检测、消行后的数组重排）必须附带简洁的中文注释。

- 如果检测到当前任务涉及视觉表现，请主动确认是否符合“软萌动物风”。
